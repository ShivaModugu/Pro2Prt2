OOP Terms

All Vocab

This website covers vocabulary for OOP.

Vocab 1

Factory - a factory is an object for creating other objects
Instantiation - creation of an object referencing a class declaration an instantiated object is named and created
Object is the thing you think about first in designing a program, they are also the units of code that are eventually derived from the process.

Vocab 2

Static the variable or method marked as such is available at the class level
Fixture the set of preconditions or state needed to run a test
Namespace a declarative region that provides a scope to the identifiers

Vocab 3

Constructor a special type of subroutine called to create an object
Facade an object that serves as a front-facing interface masking more complex underlying or structural code
Property a special sort of class member, intermediate in functionality between a field and a method.

More Vocab

Here are more important terms.

Vocab 4

Class Method methods that are called on a class rather than an instance.
Method a procedure associated with a message and an object
Property a special sort of class member, intermediate in functionality between a field (or data member) and a method.

Vocab 5

Type Hint providing hints to function to only accept the given data type
Type Cast a method of changing an entity from one data type to another
Unit Test a specification of the inputs, execution conditions, testing procedure, and expected results that define a single test to be executed to achieve a particular software testing objective

Vocab 6

Design Patterns reusable solutions to commonly occurring problems
Parent a parent is one class, and a child is another class that inherits all of the attributes and functions assigned to the parent class.
Child another class that inherits all of the attributes and functions assigned to the parent class.
